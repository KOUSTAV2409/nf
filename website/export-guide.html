Export notes

Every note in nf has a number — you see it in nf list. Export pulls note text into plain .txt files in the folder you are in now.

One note

$ nf list
 1  2026-06-01  docker system prune -af
 2  2026-06-02  refactor auth before friday
 3  2026-06-03  ss -tulpn | grep LISTEN

$ nf export 3
Exported note 3 to note-3.txt

$ cat note-3.txt
ss -tulpn | grep LISTEN

Creates note-<number>.txt with the note text only (no date line).

Several notes

Pass more than one number. Each gets its own file.

$ nf export 1 3 5
Exported note 1 to note-1.txt
Exported note 3 to note-3.txt
Exported note 5 to note-5.txt

Order does not matter. Invalid numbers print an error; valid ones still export.

All notes at once

One file, full backup — every line exactly as stored (date + text).

$ nf export all
Exported all notes to notes-2026-06-05.txt

Filename includes today’s date. Same content as ~/.local/share/nf/notes.

Quick reference

  • nf export 5 — one note → note-5.txt
  • nf export 2 7 10 — three notes → three files
  • nf export all — whole notebook → notes-YYYY-MM-DD.txt

Good to know

  • Numbers follow list order. After a delete, run nf list again.
  • Same numbers as nf del. What you see in the list is what you pass in.
  • Re-running overwrites. Exporting the same number again replaces that file.
  • Plain text only. No PDF, no cloud — files you can grep, email, or back up.
One note, a few notes, or everything — same command, different arguments.
